# 连连看项目说明与验收文档

> 版本：1.0.5.6

## 一、项目概述

LLK 是基于 **Qt6 + C++** 开发的连连看桌面游戏，采用 Visual Studio 工程管理（具体工程文件包含解决方案文件与项目文件），整体功能完整、结构清晰，涵盖多游戏模式、主题切换、音频管理、排行榜持久化、无边框玻璃风 UI 等核心功能，同时支持游戏配置自定义、数据持久化存储等拓展功能，可满足课程设计、作品展示及功能验收等多种使用场景。

## 二、技术栈与工程配置

- 开发语言：C++，遵循 C++11 及以上标准，保证代码的兼容性与可读性，适配 Qt6 框架的 API 调用规范。

- UI 框架：Qt v6.11.0，主要用到 core（核心功能）、gui（图形界面）、widgets（标准控件）、multimedia（音频播放）四个核心模块，支撑游戏的界面渲染、交互逻辑与音频输出。

- 构建环境：Visual Studio 2022 v17.14.29 + Qt VS Tools 插件，Qt VS Tools 用于实现 Qt 项目与 Visual Studio 的无缝集成，支持代码编辑、编译、调试一站式操作，确保项目可正常构建与运行。

- 工程文件：`LLK_Refresh.sln`（解决方案文件，用于管理整个项目的工程结构）、`LLK_Refresh.vcxproj`（项目文件，包含项目的编译配置、依赖项等）、`LLK_Refresh.vcxproj.filters`（筛选器文件，用于分类管理项目中的源文件、头文件、资源文件等）。

- 资源管理：`res.qrc`（Qt 资源文件，统一管理游戏所需的所有资源，包括图片、音效、背景音乐等，确保资源可被项目正确引用）、`version.rc`/`resource.h`（Windows 平台专用的资源文件，用于设置应用程序图标、版本信息、版权信息等，使游戏符合 Windows 应用的规范）。

- 运行环境：
  - Windows OS 10以上版本
  - Qt 6.0+
  - Visual C++ 14运行库

## 三、项目模块划分

### 3.1 入口与主框架

- `main.cpp`：整个应用程序的入口文件，负责执行程序启动的初始化操作，包括应用程序图标的设置、主题与背景音乐的初始化、主窗口的创建与显示，是程序启动的核心入口，确保程序从启动到主界面展示的流程顺畅。

- `mainwindow.h/.cpp`：主菜单窗口的头文件与源文件，负责搭建主界面的布局与控件，提供五大游戏模式的入口按钮，以及帮助、设置、关于、排行榜等功能对话框的调用入口，是用户与游戏交互的首要界面，统筹管理所有游戏功能的跳转。

### 3.2 核心逻辑层

- `gamelogic.h/.cpp`：游戏核心逻辑的头文件与源文件，承担连连看游戏的核心算法实现，具体包括棋盘的生成（按不同模式的规格生成成对的图案）、两个图案的连通性判断、连通路径的搜索、提示功能的实现、棋盘重排逻辑、无解状态的检测等，是整个游戏玩法的核心支撑，确保游戏规则的正确执行。

### 3.3 游戏窗口层

- `GameWindow`：所有游戏模式窗口的抽象基类，定义了游戏窗口的通用接口与基础功能，包括棋盘的绘制、用户点击事件的处理、提示与重排功能的调用、音频与主题的关联等，所有具体游戏模式窗口均继承此类，实现代码复用与统一管理。

- `GameControl`：游戏控制层（继承自 QObject），位于 UI 层（GameWindow）与逻辑层（GameLogic）之间，负责管理游戏状态机（选中/消除/提示/重排等）、处理用户点击事件分发、协调 GameLogic 的算法调用并通过信号槽机制向 GameWindow 通知状态变更，是实现 UI 与逻辑解耦的核心中间层。

- `ClassicWindow`：经典模式窗口，继承自 GameWindow，实现经典模式的专属逻辑，包括无倒计时的游戏规则、通关后的用时统计、昵称输入与排行榜记录等，是游戏的核心模式之一。

- `RelaxWindow`：休闲模式窗口，继承自 GameWindow，主打轻松游玩的体验，采用较小的棋盘规格，无倒计时限制，不记录排行榜，满足用户休闲娱乐的需求。

- `TimedWindow`：计时模式窗口，继承自 GameWindow，核心特点是带有倒计时功能，玩家需要在规定时间内消除所有图案，胜利后记录剩余时间到排行榜，同时支持暂停、继续等操作，增加游戏的挑战性。

- `LevelWindow`：关卡模式窗口，继承自 TimedWindow，在计时模式的基础上增加关卡递进机制，共设三关，每关的棋盘规格逐步增大、难度逐步提升，通关后自动进入下一关，且关卡解锁状态会持久化存储，下次启动游戏可直接选择已解锁关卡。

- `CustomWindow`：自定义模式窗口，继承自 GameWindow，允许用户自行设置棋盘的行数与列数，核心限制为行数与列数的乘积必须为偶数（确保所有图案都能成对出现，保证游戏可正常通关），满足用户的个性化游玩需求。

### 3.4 管理模块（单例）

- `ThemeManager`：主题管理模块，采用单例模式设计（整个程序中仅存在一个实例），负责管理游戏的所有主题资源，包括主题的切换、主题对应的背景图、图标等资源的加载与提供，确保游戏界面的主题风格统一且可灵活切换。

- `AudioManager`：音频管理模块，采用单例模式设计，负责游戏的背景音乐（BGM）与音效（点击、消除等）的播放、暂停、音量调节、静音设置等，统一管理音频资源，确保音频播放的一致性与灵活性。

- `RankManager`：计时排行榜管理模块，负责计时模式（TimedWindow 本体）胜利后玩家记录的添加、存储、查询与展示，数据持久化存储在 JSON 文件中，支持按剩余时间降序排序，同时提供榜单清空与 CSV 导出功能。

- `ClassicRankManager`：经典排行榜管理模块，负责经典模式（ClassicWindow）通关后玩家记录的添加、存储、查询与展示，数据持久化存储在 JSON 文件中，支持按用时升序排序，同样提供榜单清空与 CSV 导出功能。

### 3.5 UI 对话框

- `SettingsDialog`：设置对话框，用于提供游戏的各项配置选项，包括音频（音量、静音）、主题切换、游戏规则（自动重排、计时模式时长）等，用户设置后会通过 QSettings 持久化存储，重启游戏后仍保持设置。

- `HelpDialog`：帮助对话框，用于展示游戏的玩法规则、操作说明、常见问题等内容，帮助用户快速了解游戏的玩法与注意事项，提升用户体验。

- `AboutDialog`：关于对话框，展示游戏的开发信息、版本号、版权信息、最后更新日期等内容，方便用户了解项目的基本信息。

- `ResultDialog`：结算对话框，用于展示游戏的结算结果（胜利/失败）、用时/剩余时间等信息，是游戏结束后向用户反馈结果的核心界面，同时关联排行榜的记录逻辑。

- `LevelSelectDialog`：关卡选择对话框，用于关卡模式（LevelWindow）的关卡选择，展示已解锁的关卡与未解锁的关卡，用户可选择已解锁关卡开始游戏，未解锁关卡不可点击。

- `RankSelectDialog`：排行榜类型选择对话框，提供计时排行榜与经典排行榜的选择入口，用户可通过该对话框选择需要查看的排行榜类型，跳转至对应的排行榜展示界面。

- `RankDialog`：计时排行榜展示对话框，用于展示计时模式的玩家记录，按剩余时间降序排列，支持榜单清空与 CSV 导出功能，界面风格与整体玻璃风 UI 保持一致。

- `ClassicRankDialog`：经典排行榜展示对话框，用于展示经典模式的玩家记录，按用时升序排列，支持榜单清空与 CSV 导出功能，与 RankDialog 界面风格统一。

- `GlassMessageBox`：统一玻璃风消息框，用于游戏中的各类提示（如非法输入、确认关闭等），采用玻璃风样式，与整个游戏的 UI 风格保持一致，确保界面的统一性。

### 3.6 通用工具

- `uistyle.h`：玻璃风 UI 样式工具类，统一定义游戏中所有控件（按钮、卡片、标题、标签等）的玻璃风样式，包括背景透明度、边框圆角、阴影效果等，确保整个游戏的 UI 风格统一、美观。

- `windowdrag.h`：无边框窗口拖动工具类，实现游戏所有无边框窗口的拖动功能，通过监听鼠标事件（按下、移动、释放），实现窗口的自由拖动，同时避免控件误拖动，确保交互的合理性。

- `windowround.h`：圆角窗口辅助工具类，用于设置游戏窗口的圆角效果，配合玻璃风样式，提升窗口的美观度，使游戏界面更具质感。

## 四、类关系结构

### 4.1 继承关系

```Plain Text
QObject
├─ ThemeManager (单例)
├─ AudioManager (单例)
├─ GameControl (控制层，持有 GameLogic*)
│
└─ QWidget
   ├─ QMainWindow
   │  └─ MainWindow
   ├─ GameWindow (抽象基类，持有 GameControl*)
   │  ├─ ClassicWindow
   │  ├─ RelaxWindow
   │  ├─ TimedWindow
   │  │  └─ LevelWindow
   │  └─ CustomWindow
   └─ QDialog
      ├─ HelpDialog/AboutDialog/SettingsDialog/ResultDialog
      ├─ LevelSelectDialog/RankSelectDialog
      └─ RankDialog/ClassicRankDialog

GameLogic (普通 C++ 类，无 Qt 基类，被 GameControl 持有)
RankManager / ClassicRankManager (普通 C++ 类，单例模式，无 Qt 基类)
```

说明：QObject 是 Qt 中所有对象的基类，ThemeManager、AudioManager 与 GameControl 均继承自 QObject（其中 ThemeManager 与 AudioManager 采用单例模式，确保全局唯一）；GameControl 作为控制层中间类，持有 GameLogic* 指针负责调用核心算法，同时通过信号槽机制与 GameWindow 通信；GameLogic 是普通 C++ 类，不继承任何 Qt 基类，专注于连连看算法实现；RankManager 与 ClassicRankManager 同样为普通 C++ 类，采用单例模式管理排行榜数据。QWidget 是所有可视化控件的基类，MainWindow 继承自 QMainWindow（QWidget 的子类），负责主菜单界面；GameWindow 作为抽象基类继承自 QWidget，内部持有 GameControl* 指针实现逻辑代理；所有具体游戏模式窗口均继承自 GameWindow；QDialog 是所有对话框的基类，各类功能对话框均继承自 QDialog，确保对话框的统一管理与交互逻辑一致性。

### 4.2 依赖（has-a / uses）关系

- **GameWindow**：持有 `GameControl*`（m_control）指针，通过该指针间接调用 GameLogic 的核心算法功能（如连通判断、路径搜索等），实现 UI 与逻辑的彻底解耦；GameControl 通过信号槽机制（cellSelected、pairMatched、gameCleared 等信号）向 GameWindow 通知所有状态变更；同时依赖 ThemeManager 加载当前主题的图标与背景资源，依赖 AudioManager 播放点击、消除等音效，依赖 WindowDragState 实现无边框窗口拖动，是游戏模式窗口的核心依赖关系。

- **TimedWindow**：持有 `QTimer*`（用于实现倒计时功能，每秒触发一次时间更新）、`QLabel*`（用于显示剩余时间文本）和 `QProgressBar*`（timeBar，用于可视化展示剩余时间进度条）；同时依赖 RankManager，在计时模式本体（非关卡模式）胜利时，将玩家的昵称与剩余时间添加到计时排行榜中，完成排行榜记录。

- **ClassicWindow**：依赖 ClassicRankManager，在经典模式通关后，将玩家的昵称与通关用时添加到经典排行榜中，实现经典模式的排行榜记录功能，确保玩家的成绩可持久化存储与展示。

- **RankDialog** ↔ RankManager：双向依赖关系，RankDialog 依赖 RankManager 获取计时排行榜的所有记录并展示，RankManager 依赖 RankDialog 向用户展示排行榜数据，同时接收用户的榜单清空、导出等操作指令；**ClassicRankDialog** ↔ ClassicRankManager：与上述双向依赖关系一致，分别对应经典排行榜的展示与管理。

- **SettingsDialog**：依赖 AudioManager 实现音量调节、BGM 静音、音效静音等设置的生效与保存；依赖 ThemeManager 实现主题切换功能，切换后即时更新界面主题；依赖 QSettings 实现各项配置（自动重排、计时时长等）的持久化存储，确保设置重启后仍有效。

- **MainWindow**：依赖 ThemeManager 加载主界面的背景资源，确保主界面主题与整个游戏保持一致；依赖各游戏窗口（ClassicWindow 等），通过创建对应的窗口实例实现游戏模式的切换；依赖各功能对话框（SettingsDialog 等），通过调用对话框实例实现帮助、设置等功能的跳转。

## 五、功能说明

### 5.1 游戏模式

1. **经典模式**：采用 9×14 的棋盘规格（共 126 个格子，63 对图案），无倒计时限制，玩家可自由控制游戏节奏，通关条件为消除棋盘上的所有图案，通关后会自动计算玩家的通关用时，提示用户输入昵称，输入完成后将昵称与用时记录到经典排行榜中，供后续查看。

2. **休闲模式**：采用 8×8 的棋盘规格（共 64 个格子，32 对图案），无倒计时限制，不记录排行榜，主打轻松、无压力的游玩体验，适合用户碎片化时间娱乐，通关条件为消除所有图案，通关后直接返回主菜单。

3. **计时模式**：采用 8×12 的棋盘规格（共 96 个格子，48 对图案），带有倒计时功能，倒计时时长可通过设置对话框自定义，玩家需要在倒计时结束前消除所有图案，胜利后记录剩余时间到计时排行榜；若倒计时结束未消除所有图案，则判定为失败，不记录排行榜，无论胜负都会弹出结算对话框，展示结果后返回主菜单。

4. **关卡模式**：继承计时模式的核心逻辑，共设三关，关卡难度逐步提升，具体规格为：第一关 8×7（56 个格子，28 对图案）、第二关 9×10（90 个格子，45 对图案）、第三关 10×13（130 个格子，65 对图案）；通关当前关卡后，自动解锁下一关并持久化解锁状态，下次启动游戏可直接选择已解锁关卡；每关均有倒计时限制，胜负判定与计时模式一致，不单独记录排行榜。

5. **自定义模式**：允许用户自行输入棋盘的行数与列数，输入范围需符合游戏规则（行数与列数均为正整数，且行数×列数为偶数），若输入非法（如行数×列为奇数、输入非整数等），会弹出提示消息，阻止游戏启动；游戏无倒计时，不记录排行榜，通关条件为消除所有图案，通关后返回主菜单。

### 5.2 公共玩法规则

- 消除规则：只有两个相同图案的格子才可进行消除，且两个格子之间的连通路径最多允许有 2 个拐点，同时支持绕棋盘外圈连通（即路径可延伸至棋盘外部，绕外圈实现连通），不符合该规则的两个格子无法消除。

- 辅助功能：支持提示功能，点击提示按钮后，系统会自动查找棋盘上可消除的一对图案并高亮显示，方便玩家继续游戏；支持重排功能，点击重排按钮后，系统会将棋盘上剩余的图案随机打乱，重新排列；支持无解自动重排（可在设置对话框中开启/关闭），当系统检测到当前棋盘无任何可消除的对子时，会自动触发重排，直到棋盘有解。

## 六、核心算法（GameLogic）

- 地图生成：根据当前游戏模式的棋盘规格，生成成对的图案数据（确保每个图案都有且仅有一个配对），然后通过随机打乱算法，将成对的图案随机分布在棋盘的各个格子中，确保每次游戏的棋盘布局都不相同，提升游戏的趣味性。

- 连通判断：采用广度优先搜索（BFS）算法实现两个格子的连通性判断，搜索过程中记录当前位置、移动方向以及拐点数，严格限制拐点数不超过 2 个，同时支持绕外圈连通（即允许路径延伸至棋盘边界外，绕外圈寻找连通路径），确保判断结果的准确性与规则的一致性。

- 提示功能：采用按图案值分桶的策略，将棋盘上所有格子按图案类型分类，然后在每个图案类型的桶中，依次查找可连通的对子，找到第一对可连通的对子后，停止搜索并返回该对子，用于界面高亮提示，确保提示功能的高效性。

- 重排策略：通过 `shuffleUntilSolvable` 函数实现，该函数会循环调用 `shuffleMap`（随机打乱棋盘图案）函数，每次打乱后检测棋盘是否有解，若有解则停止循环，若无解则继续打乱，直到棋盘有解，确保重排后游戏可正常进行，避免出现无解导致游戏卡顿。

- 解缓存：在检测棋盘是否有解（`hasAnySolution` 函数）时，使用缓存机制存储检测结果，避免对同一棋盘布局进行重复检测，减少计算量，提升游戏的运行效率，尤其是在重排过程中，可显著减少等待时间。

## 七、UI 与交互

- 统一玻璃风 UI：通过 uistyle.h 工具类，统一设置所有控件的玻璃风样式，包括窗口背景的半透明效果、控件的圆角边框、阴影效果等，使整个游戏的界面风格统一、美观，提升用户的视觉体验。

- 无边框窗口拖动：通过 windowdrag.h 工具类，实现所有游戏窗口（主窗口、游戏模式窗口、对话框）的无边框拖动功能，监听鼠标按下、移动、释放事件，确保窗口可自由拖动，同时避免控件被误拖动（仅窗口顶栏可拖动），保证交互的合理性。

- 主题切换实时生效：通过 ThemeManager 管理所有主题资源，当用户在设置对话框中切换主题时，ThemeManager 会立即加载新主题的背景图、图标等资源，同时更新所有打开窗口的界面样式，无需重启游戏，实现主题切换的实时生效，提升用户体验。

- 对话框统一自定义顶栏：所有对话框（HelpDialog、SettingsDialog 等）均采用自定义顶栏，顶栏包含对话框图标、标题以及关闭按钮，顶栏样式与玻璃风 UI 保持一致，同时关闭按钮支持点击关闭对话框，部分对话框（如计时模式窗口）还支持最小化、最大化功能，确保对话框的交互一致性。

## 八、数据持久化

基于 `QSettings` 与 JSON 文件实现数据持久化，确保用户的配置信息、排行榜数据、关卡解锁状态等在游戏重启后仍能保留，具体分为配置项与排行榜两类数据，存储位置符合 Windows 应用的规范（排行榜文件存储在 AppData 目录下）。

### 8.1 配置项

- 主题配置：存储当前选择的游戏主题（如 classic、fruit 等），重启游戏后自动加载该主题；音频配置：存储当前的音量大小（0~100）、BGM 静音状态、音效静音状态，重启游戏后保持当前的音频设置。

- 游戏规则配置：存储自动重排功能的开启/关闭状态（开启后，棋盘无解时自动重排）、计时模式的总时长（用户可自定义，单位为秒）；关卡配置：存储关卡模式的最高解锁级别（如已解锁到第三关），重启游戏后可直接选择已解锁关卡。

### 8.2 排行榜

- `timed_rank.json`：计时排行榜数据文件，存储计时模式（本体）胜利后玩家的记录，每条记录包含昵称与剩余时间，数据按剩余时间降序排列（剩余时间越长，排名越靠前），确保排行榜的合理性。

- `classic_rank.json`：经典排行榜数据文件，存储经典模式通关后玩家的记录，每条记录包含昵称与通关用时，数据按用时升序排列（用时越短，排名越靠前），符合经典模式的玩法特点。

- 支持 CSV 导出：两个排行榜均支持将当前的记录导出为 CSV 文件，导出的 CSV 文件包含昵称、时间（剩余时间/用时）等信息，可用 Excel 等工具打开查看，方便用户备份或分享排行榜数据。

## 九、音频与主题机制

### 9.1 音频（AudioManager）

- 音频播放组件：背景音乐（BGM）采用 `QMediaPlayer + QAudioOutput` 组合实现，支持 BGM 的循环播放、暂停、继续，同时可调节 BGM 的音量；音效（点击、消除等）采用 `QSoundEffect` 实现，支持音效的快速播放，确保音效与用户操作同步。

- 音效类型：包含点击格子音效（用户点击棋盘格子时播放，使用 `playClickSfx()`）、消除成功音效（两个格子成功消除时播放，使用 `playClearSfx()`），两种音效均采用 `QSoundEffect` 实现并预创建复用对象以优化性能，确保音效与用户操作同步。

- 全局单例与配置持久化：AudioManager 采用单例模式设计，确保整个游戏中只有一个音频管理实例，避免音频播放冲突；支持音量调节（0~100）、BGM 静音、音效静音等设置，所有设置通过 QSettings 持久化存储，重启游戏后仍保持当前的音频状态。

### 9.2 主题（ThemeManager）

- 主题类型：支持多种主题，包括 classic（经典主题，12 种方块图标）、fruit（水果主题，20 种方块图标）、win（Windows 风格主题，12 种方块图标）、star（星星主题，18 种方块图标）等，每个主题对应一套独立的资源（背景图、图标等），用户可在设置对话框中自由切换；不同主题的方块图标数量不同，GameControl 会根据 ThemeManager 提供的 `tileTypeCount()` 动态设置图案类型上限。

- 资源管理：ThemeManager 统一管理所有主题的资源，包括主界面背景图路径、游戏界面背景图路径、对话框背景图路径、方块图标路径映射（按主题的图案类型，映射对应的图标文件），确保主题切换时，所有界面元素都能同步更新，保持主题的统一性。

## 十、核心流程

### 10.1 通用消除流程

1. 用户点击棋盘 → `GameWindow::onCellClicked(x,y)`：用户用鼠标点击棋盘上的某个格子，触发 GameWindow 的 onCellClicked 函数，该函数接收点击格子的坐标（x,y），作为后续处理的基础。

2. 记录选中状态 → 选两张后调用 `GameLogic::isConnected`：函数首先记录当前点击格子的选中状态，包括上一次选中的格子坐标（lastPoint）、是否已选中一张格子（hasSelected）；若当前已选中一张格子，再次点击另一张格子后，触发 GameLogic 的 isConnected 函数，判断这两张格子是否可连通。

3. 可连通：取路径 → 逻辑消除 → 刷新 UI → 绘制连线 → 模式回调 → 清盘结算：若两张格子可连通，首先调用 GameLogic 的 findPath 函数，获取两张格子之间的连通路径；然后调用 setMapData 函数，将这两张格子的状态设为 EMPTY（逻辑上消除）；接着调用 GameWindow 的 refreshCells 函数，刷新棋盘 UI，显示消除后的状态；再调用 drawLinkPath 函数，在棋盘上绘制连通路径，提示用户消除成功；随后触发派生类的钩子函数（如 TimedWindow::onPairRemoved），执行对应模式的后续逻辑；最后判断棋盘是否已清空，若清空则调用 onGameCleared 函数，进入游戏结算流程。

4. 不可连通：重置选择：若两张格子不可连通，函数会清理这两张格子的高亮状态（若已高亮），同时重置选中状态（hasSelected 设为 false，lastPoint 清空），等待用户下一次点击。

### 10.2 提示与重排流程

点击提示 → `findHintPair` → 高亮对子 / 触发 `shuffleUntilSolvable`：用户点击提示按钮后，触发 GameWindow 的 showHint 函数，该函数调用 GameLogic 的 findHintPair 函数，查找棋盘上可消除的一对对子；若找到可消除对子，立即在 UI 上高亮显示这对对子，提示用户；若未找到可消除对子（即棋盘无解），则根据设置对话框中的配置，若开启了自动重排功能，会触发 shuffleUntilSolvable 函数，循环打乱棋盘直到有解；若未开启自动重排功能，则仅提示用户当前无解，不进行重排。

### 10.3 计时模式流程

`startGame` → 定时器倒计时 → 消除对计数 → 时间到/清盘 → 结算/记榜 → 返回主界面：用户在主菜单点击计时模式，进入 TimedWindow 后，调用 startGame 函数启动游戏；startGame 函数初始化定时器（QTimer），定时器每秒触发一次 onTimerTimeout 函数，每次触发时将剩余时间（leftSec）减 1；用户每次成功消除一对图案，会触发 TimedWindow 的 onPairRemoved 函数，将消除对的数量（removedPairs）加 1，并判断棋盘是否已清空；若 leftSec 减至 0（时间到），则调用 finishTimedGame(false)，判定游戏失败；若棋盘清空，则调用 finishTimedGame(true)，判定游戏胜利；finishTimedGame 函数会停止定时器、禁止用户输入，若胜利且当前是 TimedWindow 本体（非关卡模式），则通过 RankManager 添加玩家记录，随后调用 ResultDialog 设置结算结果并显示，最后调用 backToMain 函数，返回主菜单。

### 10.4 经典模式流程

`startGame` 启动计时 → 清盘 → 计算用时 → 输入昵称 → 记榜 → 返回主界面：用户在主菜单点击经典模式，进入 ClassicWindow 后，调用 startGame 函数启动游戏，同时启动 QElapsedTimer（用于记录游戏用时）；用户消除所有图案后，触发 ClassicWindow 的 onGameCleared 函数，该函数停止 QElapsedTimer，计算游戏用时（usedSec）；随后弹出昵称输入框，提示用户输入昵称；用户输入昵称后，通过 ClassicRankManager 将昵称与 usedSec 添加到经典排行榜中；接着弹出结算对话框，提示用户通关用时与排名信息；最后调用 backToMain 函数，返回主菜单。
